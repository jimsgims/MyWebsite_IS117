HTTP/HTTPS Protocols

How do computers communicate with each other?

Web browsers access web pages with their web address (or URL, short for Uniform Resource Locator). Computers start talking to a DNS server (usually very far away), ask the server for a website, and said server reciprocates and delivers the website's IP address.

HTTP

Hypertext Transfer Protocol (HTTP) is the language used to communicate between web browsers and servers. Web browsers send something called a "get request" to these servers for the HTML code of their website. An example of a get request is when loading onto a website with many images and videos. A web browser has to make a get request for each separate image and video file. Many images and videos on one page makes the page load slower.

When users send information to the server, like filling out a form or a search query, then the web browser sends a "post request" in plaintext. The data attached is usually compared to what the server already knows (for example, login information). Personalized websites utilize something called "cookies," so that the server can remember the device for later usage. It's like an identification card of sorts, and can help personalize user experience.

Lastly, the way websites are graphically represented is with Hypertext Markup Language, or HTML for short. It is a computer language used to tell web browsers how a page looks.

You can learn more about HTML here

HTTPS

Hypertext Transfer Protocol Secure (HTTPS) is a protocol that ensures the security of the connection a web browser and a website create with each other. With plaintext requests from post requests, personal information could be hacked. Safe websites prevent snooping and tampering of personal data by communicating on a secure channel. The security protocols that do this are called Secure Sockets Layer (SSL) and its successor Transport Layer Security (TLS). The security of websites is usually indicated by a padlock, next to the web address in your web browser.

When websites ask web browsers for a secure connection, they present digital certificates to prove that they're the websites they claim to be. These certificates are identification for the websites, and are provided by certified authorities.

These authorities are trusted entities that verify the identities of websites and issue certificates for them. Poor or invalid certificates will have your web browser warning you about the website.
